## 摘要
一项伴随研究在2011年的NVIDIA Tesla C2075（Fermi，sm_20，6GB）上运行了35B专家混合模型，采用GPU预填充/CPU解码混合方式，因为4位模型无法放入设备内存（arXiv:2606.24031）。本报告保持硬件不变，探究一个能放入内存的模型能做什么：我们部署了MiniCPM-V-4.6，这是一个现代多模态助手，将SigLIP2视觉编码器和窗口注意力合并器（16倍视觉令牌压缩）与紧凑的混合门控增量网络骨干配对，完全在GPU上运行。三个结果。（i）一个基于实测基础的全GPU引擎：投影操作一次反量化8位权重并调用供应商SGEMM，该SGEMM仍存在于最后一个Fermi工具链中（达到FP32峰值的64%；我们手工编写的最佳GEMM达到37%，被错误地认为是上限）；对循环层进行分块delta规则重写，在归因暴露一个不良内核后，比顺序扫描快2.8倍；以及一个实测负面结果：4位权重在此处使解码比8位更慢，因为Fermi以半速率发出半字节解包移位指令。（ii）视觉侧是一个带有证明责任的移植：我们将塔、合并器和投影器转换为sm_20 CUDA，并针对本地生成的参考前向传播（整个塔1.4e-5）验证每个阶段。一个失败案例——位置嵌入桶化在精确有理平局上出现差异——推广为一条规则：索引算术中的浮点平局判定是实现定义的；应调用参考算子，不要重新实现。（iii）长上下文暴露了短基准测试忽略的O(N^2)瓶颈：在朴素注意力核中，预填充从2k令牌时的114 tok/s下降到10k时的21 tok/s；每头供应商GEMM调用写入现有分数缓冲区（零额外内存）恢复了平坦曲线（2k时408，10k时361；17倍），通过从60%深度精确检索指针验证。同样的重写将图像编码时间减少6倍，至0.93秒。该系统在1.7秒内端到端回答一个图像问题。

## Abstract
A companion study ran a 35B mixture-of-experts model on a 2011 NVIDIA Tesla C2075 (Fermi, sm_20, 6GB) as a GPU-prefill/CPU-decode hybrid, because the 4-bit model did not fit in device memory (arXiv:2606.24031). This report keeps the hardware and asks what a model that fits can do: we deploy MiniCPM-V-4.6, a modern multimodal assistant pairing a SigLIP2 vision encoder and window-attention merger (16x visual token compression) with a compact hybrid gated-delta-net backbone, entirely on the GPU. Three results. (i) An all-GPU engine built on measured foundations: projections that dequantize 8-bit weights once and call the vendor SGEMM still in the last Fermi toolchain (64% of FP32 peak; our best hand-written GEMM hit 37%, wrongly called the ceiling); a chunked delta-rule rewrite of the recurrent layers, 2.8x faster than the sequential scan once attribution exposed one bad kernel; and a measured negative: 4-bit weights make decode slower than 8-bit here, since Fermi issues nibble-unpacking shifts at half rate. (ii) The vision side is a port with a proof obligation: we translate tower, merger, and projector to sm_20 CUDA, validating every stage against a locally generated reference forward (full tower 1.4e-5). One failure, position-embedding bucketization differing on exact rational ties, generalizes to a rule: float tie-breaking in index arithmetic is implementation-defined; call the reference operator, do not reimplement it. (iii) Long context exposes an O(N^2) wall short benchmarks hide: prefill falls from 114 tok/s at 2k tokens to 21 at 10k in a naive attention kernel; per-head vendor-GEMM calls writing into the existing score buffer (zero extra memory) restore a flat profile (408 at 2k, 361 at 10k; 17x), verified by exact needle retrieval from 60% depth. The same rewrite cuts image encoding 6x, to 0.93s. The system answers an image question end-to-end in 1.7s.

## 2. 提出的方法论

### 2.1 核心思想
- **全CUDA推理引擎**：所有计算在GPU上完成，无需CPU跳跃或数据流式传输。
- **测量驱动优化**：每个设计决策基于实际测量，而非直觉。
- **逐阶段验证移植**：视觉塔不可直接运行于无AVX主机，通过强制CPU回退生成参考轨迹，逐层验证。

### 2.2 关键技术细节

#### (a) 语言骨干引擎优化
1. **预填充投影**：
   - 8-bit权重**一次性反量化为FP32**，然后调用供应商cuBLAS SGEMM（CUDA 8.0仍支持Fermi）。
   - 测量得到：手工寄存器分块GEMM仅达FP32峰值37%，而cuBLAS达到66%；反量化+cuBLAS管线达**659 GFLOPS（64%峰值）**，是逐位置GEMV的7.2倍。
2. **循环层（门控增量网络）**：
   - 将逐令牌扫描重写为**分块（intra-/inter-chunk）delta规则**：
     - 块内：构建下三角系统，通过两个小矩阵乘积和正向替换求解所有位置。
     - 块间：仅需顺序处理小型状态矩阵乘积。
   - 所有矩阵运算通过**批量小cuBLAS GEMM**实现。
   - 优化后，分块扫描较顺序递归**快2.8倍**（之前因正向替换核不良，曾慢2.4倍）。
3. **解码**：
   - 采用**8-bit整数矩阵-向量乘**，无移位/解包，有效带宽48 GB/s，贪心解码**48.6 tok/s**。
   - 4-bit权重实验：因Fermi发出半字节解包移位指令速率只有FMA的一半，导致计算受限，实际**比8-bit慢12%**，因此保留8-bit。

#### (b) 视觉塔逐阶段验证移植
- **障碍**：参考实现（框架前向）在无AVX主机上运行失败（缺少向量化指令）。
- **方法**：通过CPU能力覆盖使框架回退到遗留代码路径，生成本地参考轨迹（31个阶段输出）。
- **移植过程**：
  - 按阶段（嵌入层、27个编码器层、窗口注意力合并器、下采样投影器）编写手工sm_20 CUDA核。
  - 每个阶段与参考输出匹配至约10^-3（实际达10^-4或更好）后才继续。
- **关键发现**：位置嵌入桶化在精确有理平局（如10/28 = 25/70）上出现实现定义差异。结论：**这种索引算术不能重新实现，应直接调用参考算子**。

#### (c) 长上下文注意力重写
- **问题**：朴素逐位置注意力核在2k令牌时吞吐114 tok/s，到10k时骤降至21 tok/s（O(N²)）。
- **解决方案**：每头使用供应商GEMM计算分数矩阵，利用**零分配内存技巧**：现有注意力分数缓冲区（行主序 position×head）可作为GEMM输出（列主序 head×seq），无需额外开辟空间。
  - 无因果注意力：两个GEMM（QK^T, PV）后接列Softmax。
  - 因果注意力：矩形GEMM后masked softmax，尾部填充零，后续GEMM自动忽略零。
- **效果**：
  - 语言预填充：10k令牌从479s降至27.7s（**17倍加速**），吞吐量平坦（2k:408→10k:361 tok/s）。
  - 视觉塔编码：1024补丁图像从5.7s降至0.93s（6倍加速）。
  - 针检索测试精确通过。

### 2.3 公式或算法流程（文字说明）
- **分块delta规则**：给定块内C个令牌，首先计算令牌-令牌交互矩阵（小GEMM），然后通过正向替换解出所有状态更新，最后聚合输出。块间状态传递为小矩阵乘积。
- **GEMM注意力**：`S_h = Q_h * K_h^T`（完整矩形，或因果掩码后修正），然后`P_h = softmax(S_h)`，最后输出`O_h = P_h * V_h`。所有操作使用cuBLAS，每头独立执行。

## 6. 主要结论与发现

1. **Fermi的性能天花板可被供应商cuBLAS显著超越**：手工GEMM仅37%峰值，cuBLAS达66%，反量化+cuBLAS达64%，是手工方案的近2倍。

6. **端到端可行性**：2011年GPU可在1.7秒内回答图像问题，并支持多切片、多图像、多轮对话等现代特性。
