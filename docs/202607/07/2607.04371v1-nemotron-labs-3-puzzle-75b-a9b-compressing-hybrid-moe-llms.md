## 摘要
我们提出了Nemotron-Labs-3-Puzzle-75B-A9B，这是一种针对交互式部署优化的Nemotron-3-Super压缩变体。我们设计该模型以在高用户吞吐量约束下最大化服务器吞吐量。在单个8xB200节点上的交互式服务工作负载中，Puzzle-75B-A9B在匹配的用户吞吐量约束下实现了大约2倍于Nemotron-3-Super的服务器吞吐量。在单个H100 GPU上的超长上下文部署中，压缩模型将1M令牌并发数从1个请求增加到8个请求。Puzzle-75B-A9B采用多阶段流水线构建，结合了迭代Puzzle压缩框架与知识蒸馏、强化学习、量化和多令牌预测头。压缩过程联合优化异构MoE剪枝、活动参数预算和Mamba剪枝，以提高推理效率同时保持模型质量。我们在广泛的推理、编码、多语言、长上下文和智能体基准测试上评估了Puzzle-75B-A9B。尽管进行了大幅压缩，该模型在广泛的任务中相对父模型仍保持了强大的下游准确性。这些结果表明，大型混合MoE模型可以在保持强大下游能力的同时，大幅优化部署效率。

## Abstract
We present Nemotron-Labs-3-Puzzle-75B-A9B, a compressed variant of Nemotron-3-Super optimized for interactive deployment. We designed the model to maximize server throughput under high user throughput constraints. In interactive serving workloads on a single 8xB200 node, Puzzle-75B-A9B achieves approximately 2x higher server throughput than Nemotron-3-Super at matched user throughput constraints. In ultra-long-context deployment on a single H100 GPU, the compressed model increases 1M-token concurrency from 1 request to 8 requests. Puzzle-75B-A9B is constructed using a multi-stage pipeline that combines the Iterative Puzzle compression framework with knowledge distillation, reinforcement learning, quantization, and a Multi-Token Prediction head. The compression process jointly optimizes heterogeneous MoE pruning, active parameter budget, and Mamba pruning to improve inference efficiency while preserving model quality. We evaluate Puzzle-75B-A9B on a broad suite of reasoning, coding, multilingual, long-context, and agentic benchmarks. Despite substantial compression, the model retains strong downstream accuracy relative to the parent model across a wide range of tasks. These results demonstrate that large hybrid MoE models can be substantially optimized for deployment efficiency while maintaining strong downstream capability.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：大型混合MoE（Mixture-of-Experts）模型（如Nemotron-3-Super）虽然性能强大，但其推理成本高、内存消耗大、服务延迟高，尤其在生产环境中需同时满足严格的吞吐量、延迟和上下文长度要求。
- **核心问题**：如何在**不显著降低模型质量**的前提下，通过**后训练压缩**大幅提升部署效率（特别是交互式服务和高并发超长上下文场景）。
- **整体贡献**：提出一种多阶段压缩流水线，成功将Nemotron-3-Super（120.7B总参/12.8B活跃参）压缩为Puzzle-75B-A9B（75.3B总参/9.3B活跃参），在单8×B200节点上实现约2倍服务器吞吐量提升，在单H100上实现1M-token并发从1增至8，同时保持下游任务准确性。

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
采用“结构压缩 → 性能恢复”两阶段流水线，联合优化**异构MoE专家通道剪枝**、**活跃专家数（top-k）异质分配**和**Mamba SSM状态大小剪枝**，并通过**迭代式Puzzle框架**、**知识蒸馏**、**强化学习**和**部署导向的量化/推测解码**来恢复和提升性能。

### 2.2 关键技术细节

#### （1）Iterative Puzzle（迭代Puzzle）压缩框架
- 基于Puzzle [Bercovich et al., 2025] 的分解式NAS思想，但采用**多轮交替**策略：每轮进行适度结构化压缩（使用混合整数规划选择每层最佳配置），随后进行**短知识蒸馏恢复**，再作为下一轮起点重新计算替代评分。
- 这种方法比单步Puzzle更适应压缩过程中表示变化，改善剪枝质量。

#### （2）剪枝技术
- **中间通道剪枝**：对每个路由专家的FFN中间维度进行统一剪枝（不同层可不同大小）。
- **top-k减少**：每层允许不同的活跃专家数k（原始为22），降低计算开销。
- **Mamba SSM剪枝**：将SSM状态大小从128降至96，通过**贡献度量**（对输出幅度平均）选择保留的通道。

#### （3）恢复阶段
- **知识蒸馏（KD）**：分别在迭代压缩阶段和最终恢复阶段使用。第一阶段KD序列长度32K，恢复阶段扩至128K→512K，训练数据混合30%预训练+70%SFT，总训练token量约120B。
- **强化学习（RL）**：专注于恢复软件工程能力（SWE），分两阶段（单步工具比较、端到端沙箱RL），采用多学习率扫描和权重平均。

#### （4）量化
- **FP8（W8A8）**：针对Hopper GPU，量化MoE GEMM、Mamba线性层，KV cache FP8，SSM状态FP32。
- **NVFP4（W4A4）**：针对Blackwell GPU，更激进量化，SSM状态FP16+随机舍入。

#### （5）MTP与推测解码
- 继承Nemotron-3-Super的**共享头MTP**，通过**继续训练缓解训练-推理分布不匹配**（教师强迫 vs 自回归），使平均接受长度提升25%-30%。

## 6. 主要结论与发现

1. **迭代Puzzle优于单步**：三步迭代Puzzle在8个基准上平均提高0.57个点。
2. **压缩后性能保持良好**：尽管总参减少37.6%、活跃参减少26.9%，Puzzle-75B-A9B在推理、编码、长上下文上接近Super，仅指令跟随和部分智能体任务有较大差距。

6. **预填充-解码分离可行**：使用更轻量预填充模型仅导致轻微精度下降，可额外提升5%-7%预填充吞吐量。
