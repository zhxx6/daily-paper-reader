## 摘要
本报告介绍了我们在高效Qwen竞赛中的方法，目标是在资源受限的NVIDIA A10G GPU上实现Qwen3.5-4B的低延迟服务。我们的系统结合了量化目标模型与推测解码。为了恢复精度，我们在保留原始量化网格的同时对目标模型应用了量化感知蒸馏。为了加速解码，我们使用两阶段过程训练了专门针对量化目标模型的块扩散草稿模型：首先从高精度目标学习，然后适应低精度目标。由于草稿模型在每个推测解码步骤中都被调用，我们通过量化和滑动窗口注意力进一步降低其开销，在提高长上下文解码延迟的同时保持草稿令牌接受率。因此，我们的提交相比基线实现了平均6.978倍的加速，同时满足了所需的质量阈值，总体排名第3。我们希望这些结果为实际的LLM推理提供有用的见解。代码和资源可在 https://github.com/nota-github/adaptfm-quant-dflash 获取。

## Abstract
This report describes our approach to the Efficient Qwen Competition, where the goal is to enable low-latency serving of Qwen3.5-4B on a resource-constrained NVIDIA A10G GPU. Our system combines a quantized target model with speculative decoding. To recover accuracy, we apply quantization-aware distillation to the target model while retaining the original quantization grid. To speed up decoding, a block-diffusion drafter specialized for the quantized target model is trained using a two-stage procedure: first learning from the high-precision target and then adapting to the low-precision target. Because the drafter is invoked at every speculative decoding step, we further reduce its overhead with quantization and sliding-window attention, preserving draft-token acceptance while improving long-context decoding latency. As a result, our submission achieves a 6.978$\times$ average speedup over the baseline while satisfying the required quality thresholds, ranking 3rd overall. We hope these results provide useful insights for practical LLM inference. The code and resources are available at https://github.com/nota-github/adaptfm-quant-dflash

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- **量化目标模型 + 推测解码**：使用 INT4 量化的目标模型降低内存和计算，配合轻量级草稿模型快速生成候选 token，再由目标模型并行验证，以此实现加速。
- **同时量化目标与草稿**：对目标模型用量化感知蒸馏（QAD）恢复精度；对草稿模型用 PTQ（GPTQ）和滑动窗口注意力（SWA）降低其每次推测步的开销。

### 关键技术细节

#### 2.1 目标模型优化（QAD）
- **初始模型**：使用 AWQ INT4 检查点（对称逐组量化，组大小 128）作为起点。
- **蒸馏设置**：原始 BF16 精度的 Qwen3.5-4B 作为教师；学生从 AWQ INT4 去量化后的权重初始化，**冻结原始 AWQ 量化网格（scale）**，仅更新浮点权重。
- **损失函数**：逐 token 前向 KL 散度：
  \[
  \mathbb{E}_x\left[\sum_t \text{KL}\big(p_{\text{tea}}(\cdot|x_{<t})\,\|\,p_{\text{stu}}(\cdot|x_{<t}; Q_s(W))\big)\right]
  \]
  其中 \(Q_s(W)\) 是固定网格的伪量化函数，梯度通过直通估计器（STE）传递。
- **后续处理**：训练后将更新的 BF16 权重重新舍入到同一定量网格，获得新的 INT4 码字，最终以 **W4A16** 格式部署（权重 INT4，激活 BF16）。

#### 2.2 推测解码（DFlash 块扩散草稿模型）
- **草稿架构**：5 层 Transformer，隐层大小 2560，块大小 16，约 537M BF16 参数。
- **工作原理**：从目标模型的多个中间层提取隐藏状态，投影为紧凑的上下文特征后注入草稿模型的 KV 缓存；草稿模型预测每个块中掩码的未来 token（一个 anchor token 后跟掩码位置）。
- **两阶段训练**：
  - **阶段 1**：在 **BF16 目标模型** 上从随机初始化训练，使用该目标模型生成的 220K 对话数据。
  - **阶段 2**：在 **QAD 后的 INT4 目标模型** 上微调阶段 1 的草稿，使用 INT4 目标生成的 400K 新对话数据。目标权重从 INT4 网格去量化以保持数值对齐。

#### 2.3 草稿模型优化（PTQ + SWA）
- **草稿量化**：采用 GPTQ 对草稿模型进行 INT4 逐组量化（组大小 128），校准数据使用 256 个样本，从 400K 训练集中提取 4096 个校准块。对比发现 PTQ 几乎不降低接受长度。
- **滑动窗口注意力（SWA）**：仅对草稿模型应用，窗口大小可选 1024 或 2048。使用 **对称双向窗口**（查询两侧各一半），仅修改注意力 kernel，不进行 KV 缓存逐出。短序列自动退化为全注意力。

## 6. 主要结论与发现

1. **QAD 有效恢复精度**：INT4 AWQ 原本在 IFEval 上低于质量门限（0.8046 < 0.814），QAD 提升至 0.8285，同时 MMLU-Pro 和 GPQA-Diamond 也显著提升，满足所有质量门限。
2. **两阶段草稿训练优于直接训练**：从 BF16 目标学习后再适应 INT4 目标，平均接受长度 5.03 略高于直接训练 4.97，在 HumanEval 和 LongBench 上提升明显。
3. **草稿量化几乎无精度损失**：INT4 GPTQ 的接受长度（4.98）非常接近 BF16（5.03），且验证了 PTQ 方法间差异很小。
4. **SWA 显著降低长上下文延迟**：在 Long 设定下，W4 目标+W4 草稿的延迟从 2290ms（全注意力）降至 1966ms（SWA 2048）或 2026.5ms（SWA 1024）。窗口 1024-2048 是平衡点，更小的窗口（512/256）因接受长度下降而失效。
5. **最终系统**：在 A10G 上达到 **6.978× 平均加速**（Short: 2.26×, Medium: 4.19×, Long: 2.81×），同时满足质量门限。

## 7. 优点

- **方法创新**：同时量化目标模型和草稿模型，并用 QAD 对目标模型进行精度恢复，而非直接使用更低比特的 PTQ。
- **保留量化网格**：QAD 过程冻结 AWQ 的 scale，使得重新舍入后依然使用原始 INT4 网格，避免了重新量化的部署代价。
- **草稿模型的两阶段适应**：先在高精度目标上预训练，再迁移到量化目标，充分挖掘 BF16 先验知识，提升接受率。
- **草稿优化全面**：不仅用量化压缩，还引入 SWA 减少注意力开销，且验证了 SWA 对接受长度无负面影响（甚至可能提升）。
