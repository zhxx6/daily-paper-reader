## 摘要
条件计算可以将语言模型的质量与每token推理成本解耦，然而现有领先技术仅在单一轴线上独立作用：混合专家（MoE）稀疏化FFN，混合深度（MoD）跳过整个Transformer模块，KV缓存量化压缩注意力内存。我们认为这三个决策（注意力分辨率、专家选择和缓存位宽）是强耦合的，应当联合做出：一个罕见到需要充分注意的token，无论由哪个专家处理，都可能需要高精度缓存。我们提出TriRoute，一个在所有三个轴线上共享的轻量级控制器，对每一层的每个token发出协调策略：（i）注意力模式（跳过/局部/全局），（ii）稀疏的FFN专家集（含一个null专家以恢复MoD），以及（iii）KV缓存位宽。该控制器通过异构松弛（分类决策采用带直通估计的Gumbel-Softmax，专家采用负载均衡的top-k门控）在拉格朗日预算约束下进行端到端训练，该约束将平均计算和内存成本变为可控旋钮。我们发现在朴素联合训练中存在跨轴路由坍塌级联，即一个轴线上的坍塌会传播到其他轴线，并通过逐轴归一化和耦合感知平衡损失来解决。在从1.6亿到13亿参数的计算最优token数下的仅解码器模型上，TriRoute在匹配推理FLOPs和内存条件下帕累托优于最佳的独立MoD+MoE+KV量化组合，同时更好地保留了纯困惑度优化所侵蚀的稀有实体、代码和算术的尾部鲁棒性。事后分析揭示了可解释的结构：控制器将充分注意力和高精度缓存分配给句首位置、罕见子词和命名实体，而廉价地路由功能词。

## Abstract
Conditional computation can decouple language model quality from per-token inference cost, yet leading techniques act on a single axis in isolation: Mixture-of-Experts (MoE) sparsifies the FFN, Mixture-of-Depths (MoD) skips whole transformer blocks, and KV-cache quantization compresses attention memory. We argue these three decisions (attention resolution, expert selection, and cache bit-width) are strongly coupled and should be made jointly: a token rare enough to warrant full attention may also need high-precision caching regardless of which expert processes it. We introduce TriRoute, a single lightweight controller shared across all three axes that, for every token at every layer, emits a coordinated policy: (i) an attention mode (skip/local/full), (ii) a sparse set of FFN experts (with a null expert recovering MoD), and (iii) a KV-cache bit-width. The controller trains end-to-end via a heterogeneous relaxation (Gumbel-Softmax with straight-through estimation for categorical decisions and load-balanced top-k gating for experts) under a Lagrangian budget constraint that turns the average compute and memory cost into a controllable knob. We identify a cross-axis routing-collapse cascade in naive joint training, where collapse on one axis propagates to the others, and address it with per-axis normalization and a coupling-aware balancing loss. On decoder-only models from 160M to 1.3B parameters at compute-optimal token counts, TriRoute Pareto-dominates the best independent MoD+MoE+KV-quantization combination at matched inference FLOPs and memory, while better preserving tail-case robustness on rare entities, code, and arithmetic that pure perplexity optimization erodes. Post-hoc analysis reveals interpretable structure: the controller allocates full attention and high-precision cache to sentence-initial positions, rare subwords, and named entities, while cheaply routing function words.

#### 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：将这三个耦合的决策统一到一个框架下，由一个共享的**学习型控制器**联合做出，以期在给定的总推理预算（计算和内存）下，实现比独立调优组合更优的性能-成本帕累托前沿。
- **整体含义**：本文挑战了“为所有token提供均匀计算”和“独立优化不同计算维度”的范式，提出了一种更精细、自适应的资源分配哲学，旨在将宝贵的计算资源真正花在“刀刃”上。

#### 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想（TriRoute）**：在每个Transformer层前放置一个轻量级的**共享控制器**，该控制器为每个token输出一个联合策略，包含三个部分：
    1.  **注意力模式** $a \in \{\text{skip}, \text{local}, \text{full}\}$：决定该token如何读取过去的上下文。
    2.  **专家选择** $e$：从包含“空专家”（null expert，即跳过FFN）的一组专家中选择k个。
    3.  **KV位宽** $b \in \{2, 4, 8, 16\}$：决定该token自己的key/value以何种精度存入缓存，供未来的token读取。

- **关键技术细节**：
    1.  **异构松弛与梯度反传**：
        -   **注意力模式和位宽**：采用**直通估计器 (Straight-Through Estimator, STE)** 与 **Gumbel-Softmax** 结合的方法。在前向传播时使用离散的 $argmax$ 决策，反向传播时通过连续的Gumbel-Softmax松弛来估计梯度。
        -   **专家选择**：采用标准可微分的 **top-k softmax** 门控。
        -   **跨轴梯度平衡**：三个轴的梯度尺度差异巨大（跳跃注意力比降低位宽影响大得多）。论文提出对每个轴的梯度进行归一化（重缩放），防止共享trunk被某一个轴（如注意力）主导，使得位宽头能够有效学习。
    2.  **耦合感知的负载平衡（防止“级联坍塌”）**：
        -   **问题**：作者发现一个“级联坍塌”失效模式：如果注意力路由器为了省算力过早地坍缩到“跳过”模式，会导致输入到后续专家的残差状态方差极小，变得难以区分，从而专家路由器也随之坍缩，进而位宽路由器也坍缩到最低位宽。
        -   **解决方案**：引入两个机制。
            -   **逐轴白化**：在每个决策头前，对共享trunk的输出进行去中心化和归一化，防止一个轴的坍塌“饿死”其他轴。
            -   **边际熵下限**：增加一个损失项，惩罚任何一轴选项使用率的熵过低，确保路由决策的多样性，避免过早坍缩。
    3.  **单一预算旋钮（在线拉格朗日法）**：
        -   定义一个可微分的成本模型来衡量每个决策的FLOPs和内存消耗。
        -   不直接设置三个独立的稀疏度超参数，而是通过**在线拉格朗日乘子法**，将总的计算和内存预算 $C^\star$ 作为单一约束。当成本超出预算时，拉格朗日乘子 $\lambda$ 自动增加以惩罚高成本决策；反之亦然。这使得一个旋钮就能描绘出整个性能-成本帕累托前沿。

- **算法流程**：
    1.  将归一化后的token状态和辅助特征（位置、边界等）输入**共享控制器trunk**。
    2.  trunk输出被送入三个独立的决策头（注意力、专家、位宽）。
    3.  各头通过相应的松弛/采样方法输出离散决策（直通估计器、top-k softmax）。
    4.  根据决策执行对应的注意力计算、专家计算和KV缓存写入。
    5.  计算包含原始语言模型损失、负载平衡损失、熵损失和对数损失（z-loss）的总损失，并加上由拉格朗日乘子衡量的成本损失。
    6.  反向传播更新模型参数和拉格朗日乘子。
