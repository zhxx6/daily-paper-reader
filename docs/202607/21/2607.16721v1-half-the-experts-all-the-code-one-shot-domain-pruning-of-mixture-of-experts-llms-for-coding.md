## 摘要
最强的开源编码模型是混合专家（MoE）网络：它们的大部分规模来自大型的“专家”子网络池，其中只有少数专家对每个令牌起作用。这个池子使得这些模型无法在大多数开发者拥有的机器上运行，然而对于只需要编码帮助的用户来说，大多数专家编码的能力永远不会被调用。我们探究了在五种选择策略下，从两个不同家族的最新开源MoE模型（Qwen3.6-35B-A3B和Gemma-4-26B-A4B）中剪枝多少专家以及哪些专家，其评判方式与用户一致：根据模型是否仍能编写正确的代码。从任一模型中移除一半的专家在主要代码基准测试上没有统计上可检测的损失，并且损害几乎完全落在编码以外的能力上，这正是预期的权衡。但获胜策略在两个模型之间翻转，因此在一个家族上验证的配方不能假定在另一个家族上有效。我们进一步表明，困惑度（许多剪枝文献依赖的指标）可能将一个损坏的模型评级高于完整的模型；轻量级的微调可以恢复激进剪枝所损失的大约一半；而相对于将完整模型量化到相同内存，剪枝仅在量化需要降至每权重低于3位时获胜。五次试图推翻这一交叉的实验（失败标准事先确定：更好的校准、保护性选择、因果专家重要性、失败归因以及让每个模型通过执行反馈修复失败的智能体评估）均未成功；最后一次实验表明，单次基准测试普遍夸大了压缩的惩罚，因为一次修复回合完全消除了2位量化的惩罚。专家剪枝有效，但它需要在模型实际服务的任务上进行逐个模型的验证。

## Abstract
The strongest open-weight coding models are mixture-of-experts (MoE) networks: most of their size comes from large pools of "expert" subnetworks, of which only a few act on any token. That pool is why these models do not fit on the machines most developers own, yet for a user who only wants coding help, most experts encode abilities that will never be invoked. We ask how many experts can be removed, and which, by pruning two recent open-weight MoE models from different families (Qwen3.6-35B-A3B and Gemma-4-26B-A4B) under five selection strategies, judged the way a user would: by whether the model still writes correct code. Half the experts can be removed from either model with no statistically detectable loss on the primary code benchmark, and the damage lands almost entirely on abilities outside coding, the intended trade. But the winning strategy flips between the two models, so a recipe validated on one family cannot be assumed to work on another. We further show that perplexity, the metric much of the pruning literature leans on, can rate a broken model above an intact one; that a lightweight fine-tune recovers about half of what aggressive pruning loses; and that against quantizing the full model to the same memory, pruning wins only where quantization would have to drop below 3 bits per weight. Five attempts to overturn that crossover, with failure criteria fixed in advance (better calibration, guarded selection, causal expert importance, failure attribution, and an agentic evaluation letting each model repair its failures from execution feedback), all leave it standing; the last shows single-shot benchmarks overstate compression penalties broadly, as one repair turn erases the 2-bit quantization penalty entirely. Expert pruning works, but it demands per-model validation on the task the model will actually serve.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **研究动机**：当前最强的开源编程大模型多为混合专家（MoE）架构，其总参数量巨大（如 35B），但每步推理只激活少量专家（如 3B）。庞大的专家池导致模型无法装入大多数开发者拥有的消费级设备，而普通用户仅需编程能力，大部分专家能力（如多语言聊天、创意写作等）从未被使用。因此，核心问题是：能否通过一次性（one-shot）剪枝移除大量专家，得到仅保留编程能力的精简模型，且不损害编程性能。
- **整体含义**：论文旨在验证“面向领域的专家剪枝”在编程场景下的可行性与局限性，为实际部署提供指导。关键发现包括：剪掉半数专家可无损编程能力，但最佳剪枝策略在不同模型族间不可迁移；困惑度（perplexity）会误导评估；与量化相比，剪枝仅在极低比特（<3 bit）场景下占优；轻量微调可恢复部分损失。结论强调：专家剪枝有效但需针对具体模型和任务进行验证。

### 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：在 MoE 模型中，通过一次性收集每个专家的统计量（如路由频率、门控权重、激活范数等），根据特定重要性评分对专家进行排序，然后直接删除排名靠后的专家（包括对应的路由器行），从而压缩模型总参数量，而不影响模型结构适配常用推理框架。
- **关键技术细节**：
  - **Profiling（特征采集）**：使用校准数据集（coding + general mix）对基础模型进行一次前向传播，记录每个专家在每个 token 上的路由计数、门控权重和、激活范数的第一/二阶矩、以及门控加权激活范数（REAP 统计量）。
  - **评分标准**（五种）：
    - **REAP**：门控加权激活范数的均值。
    - **Routing mass（路由质量）**：门控权重之和。
    - **MAN（平均激活范数）**：激活范数的均值。
    - **Contrastive（对比评分）**：新提出，定义为目标域（编程）与参考域（通用）的 REAP 比值 \(S_{\ell,e} = \frac{F_{\ell,e}(T)}{F_{\ell,e}(R) + \varepsilon}\)，用于识别领域专长专家。
    - **Random（随机基线）**：按固定种子随机保留。
  - **Surgery（手术剪枝）**：在不加载模型的情况下，对每个 MoE 层，根据评分保留 top-K 专家（统一每层保留比例），直接通过索引选择张量维度并裁剪；同时裁剪路由器矩阵对应的行。使用合成模型验证剪枝后 logits 与屏蔽被删专家的基础模型一致。
  - **Healing（恢复微调，可选）**：对激进剪枝后的模型进行自蒸馏：由基础模型对校准提示生成回复，冻结已保留的专家，仅训练注意力、共享专家等参数（LoRA rank 64）和路由器矩阵。
  - **Export（导出）**：将剪枝后模型转换为 GGUF 格式，并应用重要性矩阵校准的量化，最终在目标设备上验证。

### 6. 论文的主要结论与发现

1. **剪枝可行性**：在编程领域，剪枝 50% 专家可在 HumanEval+ 上无统计显著损失（Qwen 上 +0.6，Gemma 上 -3.7，CI 穿过零）；MBPP+ 上 Qwen 显著损失 4.5 点，Gemma 无损失。损害集中在通用能力上（通用困惑度上升 2-6倍）。
2. **评分标准不可迁移**：Qwen 上 REAP 最优，Routing mass 在 25% 保留时崩溃；Gemma 上 Routing mass 最优，REAP 在 50% 保留时就显著落后 16.5 点。随机选择在两个模型上均崩溃。
3. **困惑度的误导性**：随机剪枝模型困惑度低于基础模型但 pass@1 低 58 点；REAP 25% 保留的代码困惑度低于 37.5% 但 pass@1 更低；Gemma 的聊天模板使困惑度膨胀 14-30 倍。结论：困惑度不能替代代码功能评估。
4. **剪枝 vs 量化**：存在明确交叉点：≥3 bit 量化时，量化完整模型更好；<3 bit 时（~13 GB），剪枝后 4-5 bit 模型显著胜出（HumanEval+ +6.7）。多次鲁棒性检验（改进校准、保护选择、因果重要性、失败归因、智能体评估）均无法推翻这一交叉。
5. **恢复微调有效**：在 25% 保留模型上，使用参数高效微调（仅调路由器和注意力，冻结专家）可恢复 HumanEval+ 上 52% 的差距（+5.4 点），与翻译领域需要全参数微调形成对比。
6. **智能体评估缩小差距**：允许一次修复回合后，所有压缩模型的性能提升大于基础模型，2-bit 量化惩罚消失，但剪枝 vs 量化的相对排序不变。
