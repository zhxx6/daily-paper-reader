## 摘要
混合专家（MoE）模型通过仅激活每个标记的一小部分专家来实现规模扩展。然而，训练这类模型仍具挑战性，因为top-k路由是离散且不可微的，需要针对专家选择的梯度估计器，其设计仍是核心开放问题。我们提出ProbMoE，一种概率路由框架，将专家选择建模为基数约束的专家子集上的分布，并将路由表述为该离散子集空间中的概率推理。我们首先提出ProbMoE Exact-k路由，其在前向传播中采样k专家子集，反向传播则使用通过每个专家精确边际概率的梯度作为真实梯度的可处理代理。ProbMoE自然推广至动态k路由设置，其中训练和推理均将路由基数约束至相同的预定义范围，从而允许每个标记的自适应专家分配。在多个基准和模型骨干上，ProbMoE Exact-k相较于竞争基线实现了强劲性能，具有更高的专家利用率和路由多样性；ProbMoE Dynamic-k以更少的激活专家达到了可比的性能。

## Abstract
Mixture-of-Experts (MoE) models scale by activating only a small subset of experts per token. However, training such models remains challenging because top-$k$ routing is discrete and non-differentiable, requiring gradient estimators for expert selection whose design remains a central open problem. We introduce ProbMoE, a probabilistic routing framework that models expert selection as a distribution over cardinality-constrained expert subsets and formulates routing as probabilistic inference in this discrete subset space. We first propose ProbMoE Exact-$k$ routing, which samples $k$-expert subsets in the forward pass, and the backward pass uses gradients through each expert's exact marginal probability as a tractable surrogate for the true gradient. ProbMoE naturally generalizes to a dynamic-$k$ routing setting, where both training and inference constrain the routing cardinality to the same predefined range, allowing adaptive expert allocation per token. Across benchmarks and model backbones, ProbMoE Exact-$k$ achieves strong performance compared to competitive baselines, with improved expert utilization and routing diversity; ProbMoE Dynamic-$k$ achieves comparable performance with fewer activated experts.

# 论文《ProbMoE: 面向混合专家模型的可微分概率路由》详细总结

## 1. 核心问题与整体含义（研究动机和背景）
- **背景**：混合专家模型（MoE）通过为每个 token 仅激活少量专家来扩展模型容量并控制计算成本。标准路由使用 hard top‑k 操作符，但该操作是离散且不可微的，导致梯度无法直接通过选择决策传播。
- **核心问题**：传统训练中忽略离散选择路径，只通过 softmax 概率传播梯度，路由缺乏对替代专家子集的探索信号，导致路由分布高度集中、专家利用率低下、训练不稳定。
- **整体含义**：ProbMoE 将路由重新定义为概率推理问题，在基数约束的专家子集空间上建模分布，并通过可处理的方式提供梯度，从而同时改善路由质量和专家利用效率。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 将每个 token 的专家选择视为从**基数约束的专家子集分布**中采样一个子集，前向传播使用采样子集计算 MoE 输出，反向传播通过专家的**精确边际概率**传递梯度。
- 自然支持两种变体：**Exact‑k**（固定专家数）和 **Dynamic‑k**（专家数可变，在给定范围 $[k_{\min}, k_{\max}]$ 内自适应）。

### 关键技术细节
- **分布定义**：对每个专家独立赋予 Bernoulli 选择概率 $p_i = \sigma(r_i)$，条件于基数 $|S|=k$ 或 $k_{\min} \le |S| \le k_{\max}$ 得到归一化分布。归一化常数可通过动态规划（DP）在 $O(Nk)$ 时间内精确计算（见定理 4.1 与 5.1）。
- **前向采样**：使用 SIMPLE（Ahmed et al., 2023）中的子集采样算法，从条件分布中采样 $k$ 个专家或从范围约束分布中先采样基数再采样子集。
- **反向传播（梯度估计）**：
  - 计算每个专家的边际概率 $m_j = \Pr(j \in S \mid \text{cardinality constraint})$，通过 $\partial \log Z / \partial \log p_j$ 得到（Prop. 5.2）。
  - 使用 Straight‑Through 估计器构造路由权重：$w = (\text{stopgrad}(z - m) + m) \odot \pi$，其中 $z$ 为采样掩码，$\pi$ 为 softmax 权重。
  - 梯度可同时通过 $\pi$ 和 $m$ 传播，确保路由收到准确的子集选择信号。
- **Dynamic‑k 具体流程**：训练时先从 $k$ 的边际分布采样基数，再从 Exact‑k 分布采样子集；推理时选择 MAP 子集（即使累计概率最大的子集）。

## 6. 主要结论与发现

### Exact‑k 路由
- ProbMoE 在多数基准上超过或匹敌最强基线（DenseMixer），尤其在 Law、Translation、MBPP 等任务提升显著（Table 1）。
- **专家利用率更高**：路由概率分布更宽，需要更多专家才能累积 99% 概率；Top‑4 质量更低、归一化熵更高，避免专家崩溃。
- **消融实验**：概率采样 + 边际梯度的组合是获得最佳性能的关键。

### Dynamic‑k 路由
- 以平均少 15–30% 的激活专家达到与 Exact‑k 可比的性能，且不同任务自适应分配：Law 平均激活专家更多，GSM 更少（Table 2、Fig. 5）。
- **Token 级自适应**：罕见 token 激活更多专家，常见 token 激活较少（Fig. 6），体现对语义复杂度的响应。
- **训练‑推理基数不匹配**：Exact‑k 训练的模型在动态推理时倾向于选更少专家，而 ProbMoE Dynamic‑k 能更准确利用容量。

### 总体
- ProbMoE 提供统一、可微、稀疏的概率路由框架，兼有理论依据和实践收益。

## 7. 优点
- **理论原则性**：将路由视为离散子集上的概率推理，归一化和边际计算可处理，梯度估计有低偏差和低方差（附录 F）。
- **自然支持动态路由**：无需启发式阈值或空专家，框架内统一处理基数变化。
- **稀疏计算保持**：前向只激活 $k$ 或 $\le k_{\max}$ 个专家，训练和推理均不引入稠密计算（对比 DenseMixer）。
- **架构无关**：可即插即用到 OLMoE、Qwen 等不同 MoE 架构。
- **丰富的实验分析**：路由多样性、自适应专家分配、消融实验、效率测量，全面地支持主张。
