## 摘要
稀疏混合专家（MoE）架构通过仅通过top-$k$路由激活一小部分专家，在固定推理预算下扩展LLM参数。尽管这保持了因果关系并适用于自回归语言模型，但离散的top-$k$算子不可微分，强制每个输入使用固定数量的激活专家，导致计算效率低下。我们提出SoftMoE，用截断的软top-$k$ LapSum松弛替代离散路由，允许基于梯度的专家路由优化。我们进一步参数化每层平均激活专家数量，并施加全局预算约束，使模型能够学习如何跨层分配专家容量。SoftMoE仍然完全兼容自回归建模，并且在语言建模和下游任务上实现了与稀疏MoE相当或更好的性能，同时激活了显著更少的专家。值得注意的是，学习到的分配高度非均匀，较后层激活更多专家。源代码已公开发布$^\dagger$。

## Abstract
Sparse Mixture-of-Experts (MoE) architectures enable scaling LLM parameters under a fixed inference budget by activating only a small subset of experts via top-$k$ routing. While this preserves causality and suits autoregressive language models, the discrete top-$k$ operator is not differentiable, forcing a fixed number of active experts per input and resulting in inefficient use of computation. We propose SoftMoE, which replaces discrete routing with a truncated soft top-$k$ LapSum relaxation, allowing gradient-based optimization of expert routing. We further parameterize the mean number of active experts per layer and impose a global budget constraint, enabling the model to learn how to allocate expert capacity across layers. SoftMoE remains fully compatible with autoregressive modeling and achieves performance comparable to or better than sparse MoE on language modeling and downstream tasks, while activating significantly fewer experts. Notably, the learned allocation is highly non-uniform, with later layers activating more experts. The source code is publicly available$^\dagger$.

## 1. 核心问题与整体含义（研究动机与背景）

- **背景**：大规模语言模型（LLM）中，稀疏混合专家（MoE）架构通过离散的 top‑k 路由仅激活部分专家，在固定推理预算下扩展模型参数。该方法虽保持自回归因果性，但存在根本性缺陷——**top‑k 算子不可微**，导致梯度无法通过选择过程传播。
- **问题**：不可微性强制每个输入使用固定数量的激活专家，无法根据输入复杂度自适应调整计算量，也无法跨层灵活分配专家预算，造成计算资源利用不均。
- **整体含义**：本文提出 **SoftMoE**，用可微的软 top‑k 松弛替代离散路由，使模型能够通过梯度优化自动学习每层应激活的专家数量，并在全局预算约束下实现计算资源的高效动态分配，同时保持自回归建模的兼容性。

## 2. 方法论

### 核心思想
- 采用基于 **LapSum 函数**的可微 top‑k 松弛（Struski et al., 2025），将离散二元路由向量替换为连续软权重 ˜p ∈ [0,1]ⁿ，满足 Σ ˜pᵢ = k（k 可为非整数）。
- 引入**截断操作**：仅保留权重高于阈值 τ 的专家，其余置零并阻断梯度，既保持稀疏性又允许输入依赖的专家激活。
- **参数化每层平均专家数**：对每层引入可学习参数 kₗ，并通过 softmax 重参数化使其满足全局预算约束 Σ kₗ = K 且 kₗ ≥ 1，从而实现跨层专家预算的端到端学习。

### 关键技术细节
- **SoftTop‑K 算子**：求解方程 LapSum(x) = Σ FLap(rᵢ − x) = k，其中 FLap 为拉普拉斯 CDF。该方程有闭式解，无需排序或迭代，复杂度 O(n)。
- **截断与路由**：输出 G(x) = T(SoftTop‑K(xW_g, k))，其中 T(z) = z·I(z > τ)。在阈值处采用次梯度约定 T′(0)=0，保持可微性。
- **跨层预算学习**：引入无约束参数 η ∈ Rᴸ，通过 π = softmax(η) 和 kₗ = πₗ·(K − L) + 1 映射到可行域，与模型参数联合优化。
- **自回归兼容性**：每个 token 独立路由，不进行 token 混合或专家侧选择，保持因果性。

### 算法流程（文字说明）
1. 输入 token 表示 x，通过门控网络生成路由分数 r = xW_g。
2. 使用 LapSum 计算可微阈值 ˜x，得到软权重 ˜p = FLap(r − ˜x)。
3. 对 ˜p 进行截断：低于阈值 τ 的权重置零，高于者的梯度正常传播。
4. 根据截断后的权重混合专家输出：y = Σ G(x)ᵢ · Eᵢ(x)。
5. 每层学习参数 kₗ 通过 softmax 重参数化受全局预算约束，与模型参数一起反向传播。

## 6. 主要结论与发现

1. **性能优于或持平稀疏 MoE**：在相同或更严格的预算下，SoftMoE* 和 SoftMoE 的语言建模损失更低，下游任务（特别是 HellaSwag）准确率更高，同时激活的专家数量显著减少（例如 ≤2 预算下减少 17%–24%）。
2. **学习到的分配高度非均匀**：在全局预算约束下，模型自发将更多专家容量分配给后层（最后 3 层占用约 50% 总预算），而浅层和中层激活更少专家。该模式在训练早期（约 25k 步）即形成。
3. **可微路由带来计算效率提升**：软路由允许模型根据输入复杂度调整激活专家数（简单 token 可能只用 1 个专家，复杂 token 用更多），而离散路由强制每 token 固定激活数。
4. **训练稳定**：软路由不引入额外训练不稳定性，动态调整激活专家数在训练中自然演化。

## 7. 优点

- **可微路由设计**：首次将 LapSum 松弛应用于 MoE，使得专家选择过程可端到端梯度优化，消除训练‑推理不匹配。
- **自适应预算分配**：通过全局约束和重参数化，实现跨层专家容量的自动学习，无需人工设定每层预算。
- **保持自回归兼容性**：每个 token 独立路由，不破坏因果掩码，可直接用于当前主流 LLM 框架。
- **计算开销低**：LapSum 仅 O(n) 复杂度（n≈32–64），相对门控矩阵乘法可忽略，不抵消激活专家减少带来的收益。
- **实证发现**：揭示了深层需要更多专家的现象，为未来 MoE 架构设计（如深度感知分配）提供了理论依据。

## 8. 不足与局限

- **语言与任务覆盖**：实验仅使用英文语料（OWT、C4）和英文下游任务，未评估多语言或多模态场景，泛化性未知。
- **模型规模有限**：总参数量 1.63B，属于中等规模，结论在更大模型（如 100B+）上是否成立有待验证。
- **截断超参数**：阈值 τ 需要根据初始激活统计手动设置，且可能影响精度与效率的权衡；探索自动化选择方法将增强实用性。
- **额外计算成本**：软路由虽然减少了平均激活专家数，但 LapSum 计算和截断操作仍引入轻微额外开销，在极低延迟场景下需评估。
- **公平性偏差风险**：软路由可能导致不同 token 被分配差异较大的计算量，可能对长尾或复杂输入产生隐性偏见，文中未深入分析。
