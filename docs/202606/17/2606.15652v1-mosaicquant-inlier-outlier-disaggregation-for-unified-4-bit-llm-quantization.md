## 摘要
4位量化显著减少内存占用并加速大语言模型推理。然而，其有限的位宽表示难以同时忠实捕捉密集的常见值（内点）和罕见的大幅度值（离群点），导致显著的精度下降。现有的混合精度方法通过将离群点保留在高精度来缓解此问题，但代价是破坏低比特执行的统一性，引入精度转换和额外数据移动，损害实际加速。我们提出MosaicQuant，一种统一的4位大语言模型量化范式，基于新颖的内点-离群点解聚原则。MosaicQuant不是提升离群点精度，而是将整个权重矩阵量化成密集的4位基分量，其中内点被忠实捕获，而离群点则不可避免地量化。然后引入一个稀疏的4位残差分量来补偿这些量化误差，选择性针对输出失真被证明最集中的误差关键权重块。然而，仅统一表示是不够的，因为将稀疏残差作为单独内核朴素执行仍然打破统一的低比特推理流水线。为弥补这一差距，我们引入ZipperEngine，它通过重叠流水线将稀疏块计算融合到密集4位GEMM内核中，不仅统一了表示，而且将执行统一为单一连贯的低比特推理流水线。在LLaMA3和Qwen3上的大量实验表明，MosaicQuant保持接近FP16的精度，同时相对于W16A16基线实现高达1.24倍的加速。

## Abstract
4-bit quantization significantly reduces the memory footprint and accelerates the inference of large language models (LLMs). However, its limited bit-width representation struggles to faithfully capture both dense common values (\emph{inliers}) and rare large-magnitude values (\emph{outliers}), causing substantial accuracy degradation. Existing mixed-precision methods mitigate this by retaining outliers in high precision, but at the cost of breaking the uniformity of low-bit execution, introducing precision conversion and extra data movement that undermine practical speedup. We propose \textbf{MosaicQuant}, a unified 4-bit LLM quantization paradigm built on a novel principle of \emph{inlier--outlier disaggregation}. Rather than elevating outlier precision, MosaicQuant quantizes the full weight matrix into a dense 4-bit base component, where inliers are captured faithfully while outlier are inevitably quantized. A sparse 4-bit residual component is then introduced to compensate for these quantization errors, selectively targeting the most error-critical weight blocks where output distortion is shown to be concentrated. However, a unified representation alone is insufficient, as naïvely executing the sparse residual as a separate kernel still breaks the unified low-bit inference pipeline. To bridge this gap, we introduce \textbf{ZipperEngine}, which fuses sparse block computation into the dense 4-bit GEMM kernel via an overlapped pipeline, unifying not only the representation but also the execution into a single coherent low-bit inference pipeline. Extensive experiments on LLaMA3 and Qwen3 demonstrate that MosaicQuant preserves near-FP16 accuracy while achieving up to $1.24\times$ speedup over the W16A16 baseline.

## 2. 方法论

### 核心思想：Inlier–Outlier Disaggregation（内点-离群点解聚）

- 首先利用等效变换（如 SmoothQuant 的迁移）将激活离群点完全迁移到权重中，得到迁移后的权重 $\hat{W}$。
- 将 $\hat{W}$ 量化为密集 4-bit 基分量 $Q(\hat{W})$，其中 inliers 被忠实表示，而 outliers 被量化后产生误差。
- 引入稀疏 4-bit 残差分量 $Q(R)$，其中 $R = \hat{W} - Q(\hat{W})$，用于补偿由离群点导致的量化误差。仅选择少量误差关键块（约 10%）进行补偿，避免全量补偿带来的额外开销。

### 关键技术细节

1. **块级补偿优于通道级**：观察到输出失真在权重的细粒度块（block）上高度集中，而非整个通道。因此采用块级残差补偿，更精确地定位高失真区域。
2. **输出感知的块选择**：直接使用输出失真（output distortion）进行块选择效果最好，但代价昂贵。为此提出 **∆-Hessian 分数**作为近似指标：
   - 对每个输入通道 $j$，使用对角 Hessian 估计 $h_j = \frac{2}{N}\sum_{n=1}^N \hat{X}_{n,j}^2$ 衡量该通道误差对输出的敏感度。
   - 对块 $B$，定义分数：$S_B = \left[\sum_{(i,j)\in B} h_j R_{ij}^2 - \sum_{(i,j)\in B} h_j (R_{ij} - Q(R)_{ij})^2\right]_+$，该分数估计了应用残差补偿后输出误差的减少量。
   - 选择 top-K 块（K 由稀疏预算决定）进行补偿。理论证明，在对角 Hessian 目标下该排序最优。
3. **算法流程**（Algorithm 1）：
   - 计算残差 $R$，估计激活敏感度 $h_j$。
   - 对每个块计算 ∆-Hessian 分数 $S_B$。
   - 选取分数最高的 K 个块，构建块掩码 $M_S$，最终量化权重为 $W_{MQ} = Q(\hat{W}) + M_S \odot Q(R)$。

### ZipperEngine：融合密集与稀疏 4-bit 内核

- **动机**：若将稀疏残差作为独立内核执行，需要重载激活、写入部分输出、再通过 reduce-add 合并，增加 30% 以上延迟。
- **融合方案**：
  - 利用稀疏块与密集 GEMM 的 tile 结构自然对齐的特点，将布局打包、量化、稀疏残差计算、reduce-add 全部融合到密集 GEMM 内核中，消除冗余 HBM 访问和内核启动。
  - **重叠流水线**：将每个 tile 拆分为独立子块，并发调度量化、密集 MMA、稀疏计算和规约等阶段（例如 Tensor Core 执行 B1 密集计算时，CUDA Core 准备 B2 的量化，HBM 预取 B3 的稀疏块），从而隐藏辅助开销，保持硬件利用率。

## 6. 主要结论与发现

1. **精度保持**：MosaicQuant 在 W4A8 和 W4A4 下均保持接近 FP16 基线的困惑度和零样本准确率，且在多个模型上优于所有对比的量化方法（包括 SpinQuant、ResQ、Atom 等强基线）。
2. **效率优势**：ZipperEngine 通过内核融合与重叠流水线，使稀疏残差补偿几乎无额外开销，在 Qwen3-8B 上比 W16A16 基线实现最高 1.24× 加速，且吞吐优于 AWQ、QuaRot、ResQ 等方法。
3. **块级优于通道级**：输出失真集中于细粒度块，块级残差补偿以更低预算实现更高效精度恢复。
4. **∆-Hessian 有效**：该分数能近似输出感知选择，且仅需一次校准排序，高效且理论最优（对角 Hessian 下）。

## 7. 优点

- **方法创新性**：提出“内点-离群点解聚”原则，将权重分解为密集基座+稀疏残差，两者均保持 4-bit，避免混合精度导致的不统一问题。

- **实验充分**：覆盖多个主流模型、多种量化位宽与基线方法，并提供了吞吐对比和消融实验，结果可信。
- **实际意义强**：适用于低精度硬件（如 RTX 4090），兼顾精度与速度，具有部署价值。
