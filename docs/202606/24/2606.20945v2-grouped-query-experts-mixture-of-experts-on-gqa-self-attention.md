## 摘要
自注意力机制是Transformer性能的核心，且在长上下文长度下通常是Transformer中最昂贵的部分，因为其成对词元交互随序列长度呈二次方增长。标准密集注意力还会对所有词元应用相同的注意力头集，无论词元难度或信息含量如何。这种均匀激活会浪费计算，特别是当序列变长且注意力成本快速增加时。我们提出了分组查询专家（GQE），一种在分组查询注意力（GQA）之上的混合专家层。在每个GQA组内，路由器为每个词元选择k个查询头专家，而所有键值（KV）头保持密集且不变。因此，GQE保持了GQA的KV缓存优势，并仅减少了活跃的查询头计算。在250M参数规模下，使用固定的300亿词元预算，GQE在激活每个词元一半查询头的情况下，与全激活GQA基线在下游准确率上相当。

## Abstract
Self-attention is central to Transformer performance and is often the most expensive part of the Transformer at long context lengths because its pairwise token interactions scale quadratically with sequence length. Standard dense attention also applies the same set of attention heads to every token regardless of token difficulty or information content. This uniform activation can waste compute, especially as sequences grow longer and attention cost increases rapidly. We propose Grouped Query Experts (GQE), a mixture-of-experts layer on top of grouped-query attention (GQA). Within each GQA group, a router selects k query-head experts per token while all key-value (KV) heads remain dense and unchanged. Thus, GQE keeps the KV cache benefits of GQA and reduces only the active query-head computation. On a fixed 30B token budget at the 250M parameter scale, GQE matches the all-active GQA baseline in downstream accuracy while activating half the query heads per token.

- **整体含义**：提出了Grouped Query Experts (GQE)，在每个GQA组内部通过路由器为每个词元选择top-k个查询头专家（query-head experts），而键值头保持密集不变。目标是实现计算效率与模型质量的平衡。

## 2. 论文提出的方法论：核心思想、关键技术细节
### 核心思想
- 在GQA框架基础上，将每个GQA组内的查询头视为一组专家（experts），路由器根据词元表示选择其中k个专家激活，其余专家跳过计算。所有KV头仍然密集计算，不改变KV缓存。
- 输出构造包含三部分：硬选择的专家输出（直接拼接）、一个通过加权求和得到的可微分路由监督槽（renormalized weighted-sum slot）、一个始终激活的共享头（shared head）。最终通过输出投影 \(W_O\) 进行线性变换。

### 关键技术细节
- **分组路由**：将N个查询头专家分为G组（G等于KV头数），每组M=N/G个专家。对于每个词元，在每个组内经过softmax后选择top-k个专家。
- **输出构造**：
  - 硬选择部分：直接拼接所选专家的输出（未加权）。
  - 加权求和槽：将所选专家的输出按组间归一化的路由概率加权求和，提供可微分的训练信号。
  - 共享头：始终计算，提供稳定锚点。
  - 三者拼接后通过缩小后的输出投影层（输入槽数从N变为kG+2）。
- **辅助损失**：添加负载均衡损失，防止路由器坍缩到少数专家。
- **计算特性**：活跃查询头占比为 (kG+1)/N，例如16头/8组k=1时，活跃9/16≈56%，其中路由专家激活8/16（50%），共享头加1。

- **精度匹配**：在30B token训练下，GQE（k=1，激活一半路由专家+共享头）在HellaSwag、ARC-Easy、PIQA上的平均准确率（56.04）与全激活GQA基线（55.86）相当，且略微超过。

## 7. 优点
