## 摘要
自注意力是Transformer性能的核心，并且在长上下文长度下通常是Transformer中最昂贵的部分，因为其成对token交互随序列长度呈二次方缩放。标准的稠密注意力还对每个token应用相同的注意力头集合，无论token的难度或信息内容如何。这种统一激活会浪费计算，特别是当序列变长且注意力成本迅速增加时。我们提出了分组查询专家（GQE），一种在分组查询注意力（GQA）之上的混合专家层。在每个GQA组内，一个路由器为每个token选择k个查询头专家，而所有键值（KV）头保持稠密且不变。因此，GQE保留了GQA的KV缓存优势，并且仅减少了活跃的查询头计算。在250M参数规模下固定的30B token预算中，GQE在下游准确率上与全活跃的GQA基线相匹配，同时每个token只激活一半的查询头。

## Abstract
Self-attention is central to Transformer performance and is often the most expensive part of the Transformer at long context lengths because its pairwise token interactions scale quadratically with sequence length. Standard dense attention also applies the same set of attention heads to every token regardless of token difficulty or information content. This uniform activation can waste compute, especially as sequences grow longer and attention cost increases rapidly. We propose Grouped Query Experts (GQE), a mixture-of-experts layer on top of grouped-query attention (GQA). Within each GQA group, a router selects k query-head experts per token while all key-value (KV) heads remain dense and unchanged. Thus, GQE keeps the KV cache benefits of GQA and reduces only the active query-head computation. On a fixed 30B token budget at the 250M parameter scale, GQE matches the all-active GQA baseline in downstream accuracy while activating half the query heads per token.

# 论文详细总结：Grouped Query Experts: Mixture-of-Experts on GQA Self-Attention

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：标准Transformer自注意力机制的计算复杂度随序列长度呈二次方增长，且对所有token不加区分地使用全部注意力头（uniform activation），导致大量计算浪费。尤其在长上下文场景下，自注意力成为主要的计算瓶颈。
- **研究动机**：希望将混合专家模型（MoE）的条件式计算思想从MLP模块迁移到注意力模块，使不同token只激活其需要的部分查询头，从而在保持模型质量的同时降低计算开销。
- **背景基础**：分组查询注意力（GQA）通过让多个查询头共享一组KV头来减少KV缓存内存和带宽，但GQA仍保持所有查询头活跃。GQE在此基础上进一步对查询头进行稀疏化路由。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：在GQA的每个组内，将查询头视为“专家”，通过路由器为每个token从每组中选择k个专家（查询头）进行激活，而KV头保持稠密（全部激活）。这样既保留了GQA的KV缓存优势，又减少了活跃的查询头计算量。
- **关键技术细节**：
  - **路由机制**：每个token在每个GQA组内，路由器生成组内专家概率分布（softmax归一化），然后选择top-k个专家。文中主要实验采用k=1，每组选1个专家。
  - **输出构建**：被选中的专家输出采用硬拼接（hard concat），不进行加权求和。额外引入一个**重新归一化的加权和槽位**（renormalized weighted-sum slot），该槽位使用路由器概率对选中专家的输出加权求和并重新归一化，为路由器提供可微分的训练信号。
  - **共享头**：设置一个始终激活的共享注意力头（shared head），提供稳定的注意力通路，辅助训练稳定性。
  - **辅助损失**：采用标准的负载均衡辅助损失，防止路由器将所有token路由到同一个专家。
  - **最终输出**：将硬拼接的专家输出、加权和槽位、共享头输出三者拼接后经过输出投影矩阵，送入后续层。由于输出投影的输入维度相对于稠密基线减少，模型参数略有变化。
- **计算剖面**：总活跃查询头数量 = kG（每组k个 × 组数G）+ 1（共享头）。在16查询头/8KV头配置下，k=1时活跃9个，占总数16的约56%；其中路由专家激活一半（8/16）。
