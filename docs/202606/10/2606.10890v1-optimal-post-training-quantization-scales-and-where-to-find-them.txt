## Abstract 

Post-training quantization (PTQ) compresses large language models by mapping weights to low-bit representations. The scaling factor that defines the quantization grid is typically chosen using simple, data-free heuristics. In this work, we present PiSO (Piecewise Scale Optimization), an algorithm that leverages calibration data to compute the optimal channel-wise weight scales exactly and efficiently under round-to-nearest quantization. PiSO partitions the scale search space into finitely many intervals on which the objective admits a closed-form minimizer. We extend PiSO to group-wise quantization via principled heuristics and propose effective strategies for interleaving scale optimization with error correction. Experiments on Llama and Qwen models across multiple model sizes and target weight bit-widths demonstrate consistent improvements in perplexity and downstream zero-shot accuracy, both standalone and combined with error correction. In particular, we observe increased benefits as the target bit-width narrows and quantization becomes more challenging. 

## 1 Introduction 

Large language models incur significant memory and compute costs that hinder deployment on resource-constrained hardware. Post-training quantization (PTQ) addresses this by mapping weights (and sometimes activations) to low-bit representations without retraining. Because quantization is inherently lossy, PTQ methods seek to minimize the resulting accuracy degradation. PTQ methods map each weight onto a quantization grid, a discrete set of representable values determined by the target low-precision format, a scaling factor, and (optionally) a zero point. Among existing PTQ techniques, greedy error correction algorithms such as OPTQ/GPTQ [ 1 ], and more recently GPTAQ [ 2] or Qronos [ 3], have emerged as the dominant approach for minimizing accuracy degradation. Crucially, these methods operate on fixed quantization grids, so the scaling factors are selected beforehand and held fixed during error correction. In PTQ pipelines, the scaling factor, or simply the scale, is typically chosen with simple heuristics: either mapping the maximum absolute weight value to the extreme grid point (the so-called absmax scale), or searching for the scale with lowest weight reconstruction error over a finite set of candidates between zero and absmax [ 3–5 ]. Both approaches are data-free : they ignore how input activations amplify different weight errors at the layer output. While some recent data-aware methods [ 6– 8]have begun to address this, they remain limited (see Section 2).

This motivates two questions: (1) can the optimal channel-wise scales be computed exactly and 

efficiently for round-to-nearest (RTN) quantizers under a data-aware reconstruction objective; and (2) can such an optimization be interleaved with established error correction algorithms such as GPTQ to jointly refine scales and weight-to-grid assignments? We answer both questions affirmatively. 

Contributions. Our main contributions are: (1) PiSO, an algorithm that efficiently computes the optimal scales for channel-wise weight quantization with respect to a data-aware layer output reconstruction objective under RTN quantization; (2) an extension of PiSO to group-wise quantiza-tion via two complementary approximations, one of which captures cross-group interactions; and (3) practical strategies for integrating PiSO with error correction methods such as GPTQ, where the RTN assumption no longer holds, by interleaving scale optimization with weight-to-grid assignments. We validate all contributions with comprehensive experiments on Llama and Qwen models across multiple model sizes and target weight bit-widths. Our results show consistent improvements in perplexity and downstream zero-shot accuracy from the use of PiSO, both standalone and combined with GPTQ and Qronos. In particular, we observe increased benefits as the target bit-width narrows and quantization becomes more challenging. Moreover, our results also suggest that PiSO reduces the calibration data required for accurate quantization, both when used with RTN and when combined with GPTQ.

Greedy rounding. These algorithms typically assume a fixed quantization grid and process weights sequentially in a given layer: each weight is assigned to a grid point, and the resulting quantization error is compensated for in the remaining unquantized weights so as to minimize the layer output error. OBC [ 11 ] introduced this framework by adapting Optimal Brain Surgeon [ 18 ] to layer-wise PTQ, and GPTQ [ 1] scaled it to LLMs through an efficient implementation. Qronos [ 3 ] and GPTAQ [ 2] are recent improvements on GPTQ whose objective accounts for the quantization error propagated from previously quantized layers. 

Scale selection. Although greedy rounding algorithms operate on a fixed grid, the choice of scale directly determines the rounding errors they must compensate [ 1]. The dominant choice in PTQ pipelines remains data-free : either absmax, which maps the largest-magnitude weight to the extreme grid point, or grid-search over a finite set of candidates in (0 , s absmax ] that minimizes the per-channel weight reconstruction error [ 3– 5]. Both ignore how input activations amplify weight errors at the layer output, and both implicitly cap s at sabsmax , a restriction that can be suboptimal (as exemplified in Figure 4 in Appendix D). The literature on data-aware scale selection for gradient-free PTQ is limited and, to the best of our knowledge, restricted to three works that propose it alongside coordinate-descent procedures for the weight-to-grid assignments: CDQuant [ 6], COMQ [ 7 ], and Beacon [ 8]. They differ in how the two are coupled. CDQuant decouples them, optimizing the scale by grid-search on the layer output error before rounding; this inherits the suboptimal s ≤ sabsmax 

constraint and the discrete granularity of the search. COMQ and Beacon instead jointly refine scales and grid assignments, an NP-hard problem on which their iterative procedures converge but offer no global-optimality guarantees. Moreover, all three methods are further limited in the group-wise setting: COMQ and Beacon are restricted to channel-wise quantization, while CDQuant’s group-wise variant treats groups independently, ignoring cross-group interactions through the activations. A complementary line of work focuses on scale selection, in quantization-aware training (QAT) [ 19 , 20 ]or gradient-based PTQ [ 13 , 16 , 17 ]; both require differentiation and lie outside the scope of this gradient-free work. 

## 3 Methods 

In this section we introduce PiSO (Piecewise Scale Optimization), an algorithm that computes provably optimal channel-wise weight scales under RTN quantization (i.e., each weight is mapped to its nearest grid point, with no error-correction reassignment). We then extend PiSO to two settings in which the optimality guarantee no longer holds: (1) we adapt it to group-wise quantization via two principled approximations; and (2) we integrate it with greedy error correction methods such as GPTQ via three strategies. We start by restricting ourselves to the RTN setting and study the optimization of the scaling parameters under this constraint. 3.3 Integration with Existing PTQ Methods 

Integration with the transform stage of the PTQ pipeline (Section 2) is immediate: one can just apply PiSO to the transformed weights produced by methods such as rotation-based equalization [ 9 , 10 ]before moving into the round stage. Integration with error correction methods such as GPTQ [ 1] and Qronos [ 3] is more delicate: these algorithms operate with a fixed scale and reassign weights away from their nearest grid point in order to compensate for errors propagated from previously quantized weights in the same channel. This conflicts with the RTN assumption underlying PiSO. Nevertheless, given the effectiveness of error correction methods, we propose three strategies to integrate them with PiSO. We focus our exposition on GPTQ, but the strategies transfer directly to other layer-wise error correction methods such as Qronos. 

Decoupled optimization. The scales of all layers are optimized first, using the ˜X produced by RTN-quantizing preceding layers with their optimized scales; error correction is then applied with the scales held fixed. The two stages are fully decoupled, but there is a mismatch between the ˜X used for optimizing the scale of a given layer and the activations that the layer actually receives when error correction is being applied, since error correction modifies first the weights of its preceding layers. 

Interleaved layer-wise optimization. Scale optimization and error correction alternate at the layer level: the scales of a given layer are optimized immediately before its error correction step. Since all preceding layers have been fully processed (scales optimized and error corrected), the ˜X used during scale optimization now matches the activations the current layer receives for error correction or at inference. Error correction itself is unchanged, as it still operates on a fixed grid. 

Interleaved group-wise optimization. In group-wise quantization, the coupling can be pushed further by optimizing the scale of each group just before error correction quantizes its weights, so that the scale is fitted on weights that already incorporate the error-diffusion updates from earlier groups in the same channel. 5 As in the layer-wise strategy, the ˜X used during scale optimization matches the activations the layer actually receives. While the weights of the group will not be quantized by RTN once the scale is fixed, the deviations from RTN are expected to be small, since each weight only needs to compensate for the error of its predecessors within the same group rather than the entire layer. This integration constrains the column processing order: error correction methods such as GPTQ typically traverse columns by decreasing diagonal-Hessian magnitude, but here all weights of a group must be processed contiguously. We therefore adopt a group-aware importance ordering: columns are sorted by Hessian importance within each group, and groups are sorted by aggregate importance (more details in Appendix A).

Calibration size sensitivity. The number of calibration samples needed by PTQ algorithms is usually overlooked. In channel-wise quantization, PiSO combined with RTN optimizes only one parameter per channel while GPTQ has to optimize all the weight-to-grid assignments. Therefore, we expect the former to be more data-efficient. To test this we sweep the calibration size on Llama-3.2-1B at 3-bit channel-wise quantization, comparing RTN + PiSO, GPTQ + absmax, and GPTQ ⊙ PiSO, with RTN + absmax included as a calibration-independent reference. The results are reported in Figure 2. As expected, RTN + PiSO is highly robust: a single calibration sample (sequence of 2048 tokens) already suffices to recover near-asymptotic perplexity and accuracy, while GPTQ + absmax requires tens of samples to stabilize. Somewhat surprisingly, GPTQ ⊙ PiSO is also more data-efficient than GPTQ + absmax despite optimizing the same number of weight assignments, plus the channel scales, suggesting that providing GPTQ with a well-calibrated grid can potentially reduce its dependence on calibration size. We leave a deeper analysis of this phenomenon, together with a broader empirical study across models, bit-widths, and PTQ methods, to future work.

## 5 Conclusion 

We revisited an often overlooked component of PTQ: the choice of the scaling factor. We introduced PiSO (Piecewise Scale Optimization), which, for channel-wise RTN weight quantization, exactly minimizes a data-aware layer-wise objective in O(D2|G|) time per channel. On top of this exact channel-wise solver we proposed two approximations for group-wise quantization and three integra-tion strategies that interleave PiSO with GPTQ/Qronos, showing that PiSO can also be leveraged outside the RTN regime, despite the optimality guarantees not carrying over. Across Llama and Qwen models, PiSO consistently improves perplexity and zero-shot accuracy over absmax, data-free heuris-tics, and Beacon, with gains widening as the bit-width narrows. Beyond accuracy, our calibration-size study (Figure 2) reveals that RTN + PiSO reaches near-asymptotic quality from a single 2048-token calibration sequence, and that GPTQ improves its data efficiency when combined with PiSO scales. We hope that our theoretical framework, and in particular PiSO, provide a foundation for treating the scale with the same algorithmic care that has been devoted to rounding decisions. 

Limitations and future work. PiSO’s exact optimality is, by construction, tied to the channel-wise RTN regime: the group-wise variants are approximations and the interleaved integration with GPTQ/Qronos breaks the RTN assumption by design. However in practice, our extensive experiments 9show that PiSO remains the strongest scale-selection strategy in both settings, and that the tightest coupling with error correction yields the best end-to-end results. Tightening the theoretical analysis of these approximations, together with a deeper empirical study of the data-efficiency phenomenon observed in Figure 2, are natural next steps. We focused on unconstrained BF16 scales; hardware formats such as MX [ 30 ] and NVFP4 [ 31 ] impose additional constraints (e.g., low-bit or hierarchical scales) that fall outside the scope of this work, and we plan to extend our framework and PiSO to such restricted-scale settings, as well as to asymmetric quantization, in future work.

## A Implementation Details 

Algorithm 1 summarizes the optimization procedure described in Section 3 for a single channel. In this implementation the sweep proceeds from large to small |s| (right to left), processing positive scales first and then negative scales, if permitted. We implemented PiSO using PyTorch 2.8 [ 32 ], and benefited from batched operations to parallelize it in GPU across channels in a given layer, and simultaneously across groups for the independent approximation (see Section 3.2). In the following, we provide additional details on how corner cases arising in the scale optimization are handled. 

Degenerate Intervals. Two transition scales coincide ( ti,ℓ = tj,ℓ ′ ) whenever wi/w j = mℓ′ /m ℓ

for grid midpoints mℓ, m ℓ′ . When this happens, the interval between them has zero length. At such a point several entries of q attempt to change simultaneously, but since our sweep processes them one at a time, the intermediate state does not correspond to any realizable rounding. We detect these degenerate intervals by checking |τj−1 − τj | < ε and skip the error update on them. 

Handling the scale sign change. For quantization grids that are symmetric around zero, i.e. they verify the property that if q ∈ G ⇒ − q ∈ G, only positive scales s > 0 need to be considered to guarantee optimality. This holds since s · q(w; s) = ( −s) · q(w; −s). However, for asymmetric quantization grids, such as those of integer datatypes, the optimal scale may be negative. If one wants to allow for such negative scales, the algorithm needs to handle the sign change at s = 0 , to account for the simultaneous changes in the unscaled grid assignments. Precisely, when the sweep crosses from positive to negative scales, the ratio wi/s flips sign for every weight simultaneously. Indeed, this is the only point where more than one entry of q changes at once, and incremental updates do not apply. We handle it by precomputing a transition vector qtr that assigns each weight to the extreme grid point matching its sign
