Abstract 

Causal tracing of factual recall has been stud-ied predominantly in dense transformer lan-guage models, where interventions localize in-formation flow to layers or feed-forward mod-ules. Sparse mixture-of-experts (MoE) lan-guage models introduce a sharper question: when a factual prediction is mediated by a routed MoE block, which routed expert con-tributions matter? We formulate expert-aware causal tracing for sparse MoE language mod-els. Using C OUNTER FACT facts, we first cor-rupt the model’s factual preference by adding noise to subject-token embeddings, and then test whether clean MoE-block outputs or clean expert-level updates restore the true-vs-foil logit contrast. For Q WEN 3-30B-A3B-B ASE ,a layer sweep selects and validates layer 44, and expert-level tracing identifies L44E069 as an expert repeatedly selected in the clean run whose held-out patch outperforms other ac-tive same-layer expert patches. For M IXTRAL -8X7B-V0.1, layer-level tracing validates a mid-layer signal, but the signal is not localized to the selected singleton expert; a coalition check instead recovers it with routed multi-expert up-dates. These results suggest that MoE factual tracing can be made expert-aware, while also showing that expert-level localization is model-and protocol-dependent rather than universal. 

1 Introduction 

Factual recall is a common setting for causal analy-sis of language models: given a prompt such as The capital of France is , the model should assign high probability to the correct object, Paris . Causal trac-ing and activation patching study this behavior by corrupting a prediction and then restoring internal components from a clean run to test which com-ponents mediate the recovery (Meng et al., 2022; Geva et al., 2023; Vig et al., 2020; Wang et al., 2023). In dense transformers, these interventions often target residual-stream activations, attention outputs, or feed-forward modules. Sparse mixture-of-experts (MoE) language mod-els make this localization problem more fine-grained. A token is not processed by a single dense feed-forward module, but by a small routed subset of experts (Shazeer et al., 2017; Fedus et al., 2022; Jiang et al., 2024; Yang et al., 2025). Thus a layer-level patch can show that an MoE block matters, but the patched block is an aggregate over routed expert updates. It does not answer which selected expert contributions support the factual prediction. This paper asks: Can factual-recall causal trac-ing be made expert-aware in sparse MoE language models? 

We study this question in a controlled diagnostic setting using filtered single-token C OUNTER FACT 

(Meng et al., 2022) contrasts. For each fact, the in-put is a subject–relation prefix and the object is not included in the prompt. We measure factual pref-erence as a true-vs-foil next-token logit difference, corrupt the run by adding noise to subject-token embeddings, and ask whether clean internal compo-nents restore the preference. Our protocol has two stages: first, we patch clean final-position MoE-block outputs into the noised run and sweep MoE layers on discovery cases; second, at the selected layer, we estimate routed expert contributions by ablation difference and patch an expert-level clean-minus-noised update into the noised run. We com-pare selected experts against other clean-active ex-perts from the same layer. The main result is asymmetric across two MoE backbones. On Q WEN 3-30B-A3B-B ASE , the layer sweep selects layer 44 and validates it on held-out C OUNTER FACT cases; expert-level tracing then selects L44E069, which validates on held-out cases and outperforms other active same-layer experts. On M IXTRAL -8 X7B-V0.1, layer-level tracing se-lects a mid-layer site, layer 19, which also vali-dates, but the selected singleton expert L19E006 underperforms active controls. A coalition check (patching either the clean-routed top-2 experts or 1

The main empirical insight is that layer-level factual recovery and single-expert localiza-tion can come apart: Qwen3 admits a robust single-expert intervention, whereas Mixtral’s validated layer-level signal is better explained by routed ex-pert coalitions. 

2 Related Work 

Factual recall and causal tracing. Language models can express factual associations through cloze-style prompts (Petroni et al., 2019). Prior work links factual knowledge to feed-forward mem-ories, knowledge neurons, and causal tracing inter-ventions (Geva et al., 2021; Dai et al., 2022; Meng et al., 2022; Geva et al., 2023; Wang et al., 2024; Liu et al., 2025). Building on causal tracing and activation patching, we use forward-pass interven-tions to test whether clean internal components can restore a corrupted true-vs-foil factual preference. 

Sparse MoE models and interpretability. 

Sparse MoE language models route each token to a small subset of experts, increasing parameter count while keeping active computation sparse (Shazeer et al., 2017; Lepikhin et al., 2021; Fedus et al., 2022; Du et al., 2022; Jiang et al., 2024; Yang et al., 2025). Recent work studies routing patterns, ex-pert specialization, and MoE knowledge attribution (Fayyaz et al., 2026; Bandarkar et al., 2026; Wang et al., 2026; Li et al., 2025; Gu et al., 2026). Be-cause the selected experts and routing weights are input-dependent, we distinguish MoE-block output patching from per-expert contribution patching. 

3 Method 

3.1 COUNTER FACT factual tracing setup 

Following the clean/corrupt intervention paradigm used in causal tracing and activation patching, we ask whether replacing a corrupted internal compo-nent with its clean counterpart restores the model’s factual preference (Meng et al., 2022; Vig et al., 2020; Wang et al., 2023). We use C OUNTER FACT 

facts introduced by Meng et al. (2022). Each record provides a subject, a relation-specific prompt tem-plate, a true object, and a counterfactual target ob-ject, which we use as the foil. For a prompt x, true object at, and foil object af , we define: 

∆( x; at, a f ) = ℓx(at) − ℓx(af ), (1) where ℓx(a) is the next-token logit for object token 

a. The object token is not included in the input; true and foil objects only define the next-token contrast. For example, if the input prompt prefix is “Virginia State Route 33 is located in” ; Virginia is the true next-token object and Milan is the foil. We apply model-specific filters for single-token true/foil objects and sufficient clean/noise margins; Appendix A gives the exact thresholds and case IDs.

3.3 MoE-block output patching 

For each clean prompt, we cache the final-position output of an MoE block at layer ℓ. This output is a hidden-state update vector produced by the routed MoE sublayer. We then run the noised prompt and replace the corresponding final-position MoE-block output with the clean one. The rescue score is: 

Rescue ℓ = ∆ patched ,ℓ − ∆noised . (4) We define rescue as a task-specific activation-restoration score: the change in the logit-difference metric after patching a clean component into a cor-rupted run. Positive rescue means that the clean MoE-block output restores part of the true-vs-foil factual preference. We use a discovery/validation split. Layers are selected only on discovery cases and then evaluated as fixed hypotheses on validation cases. 23.4 Expert-contribution patching 

At a selected MoE layer, let MoEOut( x) denote the final-position MoE-block output. For a clean-active expert e, we estimate its routed update by ablation difference: 

ce(x) = MoEOut( x) − MoEOut \e(x), (5) where MoEOut \e is obtained by suppressing ex-pert e’s routed contribution under the original top-k

routing decision. We do not reroute to a replace-ment expert nor renormalize the remaining expert weights. We define the expert patch vector 

δe = ce(xclean ) − ce(xnoised ), (6) and add δe to the noised MoE-block output at the final position. This asks whether replacing the noised routed expert update with its clean counter-part restores the true-vs-foil factual preference. Expert selection must avoid two simple con-founds. First, an expert can obtain high mean rescue by appearing in only one or two discov-ery cases. We therefore use recurrence-first selec-tion: on discovery cases, we consider clean-active experts at the selected layer, require an expert to appear in at least a fixed number of discovery cases, and then select the recurrent candidate with high-est mean rescue. Second, a selected expert may not be special if other active experts in the same layer would rescue equally well. On validation cases, the selected expert is fixed and compared with same-source active-random experts, i.e., other clean-active experts selected by the router in the uncorrupted run for the same validation prompt and MoE layer. 4 Experiments 

4.1 Models 

We evaluate two sparse MoE language models. QWEN 3-30B-A3B-B ASE has 48 MoE layers, 128 experts per MoE layer, and top-8 routing (Yang et al., 2025). M IXTRAL -8 X7B-V0.1 has 32 MoE layers, 8 experts per MoE layer, and top-2 routing (Jiang et al., 2024). 

4.2 Data and splits 

For each model, we shuffle COUNTER FACT 

records, scan records until collecting 256 usable cases, and split them into 128 discovery and 128 validation cases. The single-token and subject-noise filters are model-specific because tokeniz-ers and clean/noised logits differ across models. The recurrence threshold in Section 3.4 is set to half of the discovery split: a candidate expert must be clean-active in at least 64 of the 128 discovery cases. 

4.3 Evaluation 

Layer rescue asks whether a clean MoE-block out-put restores the corrupted true-vs-foil preference. Expert rescue asks the same question for one routed expert update. Specificity Spec asks whether the se-lected expert rescues more than other clean-active experts from the same source prompt and layer. We report mean effects, positive fraction, bootstrap confidence intervals, and two-sided sign-flip tests. Additional controls and reproducibility details are in Appendices C and A; the supplementary artifact contains scripts, row-level results, hook points, and metadata.

6 Discussion 

Block-level rescue is not expert-level evidence. 

The main methodological lesson is that MoE fac-tual tracing should not stop at layer-level activation patching. A clean MoE-block output can rescue a noised factual prompt, but this only localizes the signal to an aggregate routed block. Since the block output is composed of selected expert up-dates, expert-aware tracing asks which routed con-tributions support the recovery. Router selection identifies candidates, but selection alone is not a causal claim; causal evidence comes from patching the estimated expert update and comparing it with active controls. 

Mixtral exemplifies the limits of single-expert localization. In Qwen3, the selected layer and selected expert both validate, with multiple con-trols supporting L44E069. In Mixtral, layer-level tracing transfers to a mid-layer band, but the se-lected expert underperforms active controls. Acoalition check suggests why: clean top-2 and clean/noised routing-union expert coalitions re-cover rescue close to the L19 MoE-block effect (Appendix G). We treat these findings as model-specific, since Qwen3 and Mixtral differ in tok-enizer, training, routing design, and retained cases. 

The scope is a controlled diagnostic. We do not claim a universal factual expert or cross-model expert homology. Because tokenizer and subject-noise filters are model-specific, we treat the Qwen3 and Mixtral results as two controlled cases rather than an architecture-level comparison. The relaxed-filter scale-up preserves the same qualitative pat-tern, but the setting remains a filtered, single-token, final-position diagnostic. 

7 Conclusion 

We introduced expert-aware causal tracing for sparse MoE factual-recall diagnostics. Qwen3 shows a layer-level site that can be further localized to a robust routed expert contribution, L44E069. Mixtral shows a different pattern: layer-level res-cue transfers, but the signal is better recovered by routed expert coalitions than by the selected sin-gle expert. These results suggest that MoE factual tracing should distinguish aggregate MoE-block rescue, single-expert localization, and multi-expert routed recovery. 4Ethical Considerations 

This work uses public model checkpoints and COUNTER FACT -style factual diagnostics, and does not involve human subjects or private data. The method may support beneficial model debugging and factual-behavior analysis. A potential dual-use risk is that more precise localization of factual behavior could support targeted manipulation, sup-pression, or editing of model outputs. Such uses should be evaluated carefully before deployment, especially for factual associations involving peo-ple, groups, languages, or regions. The submitted artifact contains diagnostic scripts and derived row-level results, but no model weights or deployment tool. 

Use of AI assistance. We used GPT-5.5 (OpenAI, 2026) as a coding and limited writing assistant. The authors checked the analyses, verified results, and are responsible for all content. 

Limitations 

This study is a controlled diagnostic rather than a full account of natural factual recall. The main pro-tocol uses 256 model-specific single-token true/foil COUNTER FACT contrasts and final-position in-terventions. Although the relaxed-filter 512-case check preserves the main pattern, the filtering fun-nel may still bias retained cases toward relations and objects with clean single-token margins and subject-noise sensitivity. Expert contributions are estimated by ablation difference under fixed rout-ing and patched at the final prediction position; this does not establish a complete subject-to-answer cir-cuit. Our two-stage design decomposes the selected MoE layer rather than exhaustively searching all layer-expert pairs, so useful experts outside the selected layer may be missed. Qwen3 and Mix-tral differ in architecture, routing, tokenizer, and training, so the model contrast should be treated as two controlled case studies rather than a broad sur-vey of MoE language models. Future work should extend to more datasets, more MoE backbones, multi-token objects, subject/relation-position inter-ventions, and systematic coalition analyses across layers and models. Reproducing the full experi-ments requires large sparse MoE checkpoints and implementation-specific hooks; the artifact pro-vides scripts and row-level outputs, but the compute requirement remains nontrivial.
